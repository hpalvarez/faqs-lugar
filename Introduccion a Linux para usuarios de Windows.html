Introducción a Linux para usuarios de Windows

    En esta corta pero efectiva guía voy a explicar más o menos como instalar Linux en una máquina equipada con el sistema in-operativo Windows 98 (o 95 o 3.1 o el que se les ocurra). Claro esta, esto lo haremos sin perder esos MP3 que nos bajamos ayer de Internet o esos documentos de Word que nos matamos haciendolos. Bueno, basta de palabrerio y manos a la obra!
Primero quiero poner en claro algunas cosas que me suelen preguntar chateando en Ciudad Internet (conectados.ciudad.com.ar:6667 canal #linux).

  • Linux es una implementación gratuita de UNIX, creada por el estudiante finlandés Linus Torvalds y desarrollada alrededor del mundo por mas de 1000 hackers. Linux es compatible POSIX.1, lo que quiere decir que es parecido a los UNIX comerciales, de hecho programas desarrollados para otras plataformas compatibles POSIX.1 funcionan perfectamente en Linux.
  • Linux es 32-bit 100%, multitarea, multithreading (varias tareas dentro de una tarea). Algunas versiones de 64-bit ya estan en fase avanzada de desarrollo.
  • Estoy usando Linux 2.0.35 hace 4 meses y no tuve un solo cuelgue.
  • Corre en procesadores Intel x86 (y compatibles tipo AMD, Cyrix), Alpha, SPARC, PowerPC, Motorola 680x0, y hasta hay una versión reducida para 286 y XT!.
  • Linux puede acceder a particiones DOS, Windows (VFAT y FAT32), Windows NT (NTFS) -solo en los kernels experimentales 2.1.x- , ISO9660 (para leer los CD-ROMS), y una decena mas.
  • Para usar Linux NO hace falta saber programar en C, ni C++, ni nada. Yo no progaramo nada y he hecho con Linux miles de cosas más que en Windows.
  • Linux tiene interfaz gráfica aparte del modo texto. Se llama XFree86 y es cientos de veces mas estable que la de Windows, y hasta pueden elegir que gestor de ventanas usar (un gestor de ventanas es el programa que define el entorno de XWindows, por ej. hay un gestor que deja a XWindows igualito a Win95!)
  • Linux no es ni shareware, ni freeware, ni de dominio publico. Es software bajo licencia GNU, esto es, libremente distribuido con su código fuente, modificable, con copyright.
  • Pueden ejecutar muchos programas de DOS con DOSEmu (emulador de DOS) y muchos de Windows con WINE (emulador de Windows)
  • La cantidad de juegos para Linux no es muy alta, pero si realmente quieren jugar, dejen una particion Windows o comprense una PlayStation, que van a ahorrar horas de bajar nuevas versiones de DirectX.
  • Recuerden que Windows y Linux son muy distintos, no esperen clips con ojitos ofreciendoles ayuda.
  • Los Winmodems NO funcionan en Linux -al menos todavia-.
  • Distribución: Linux solo es el kernel - un archivo de medio mega - que es el nucleo del sist. operativo. Para ser usable necesitan mas programas, que los pueden bajar via FTP o comprarse una distribución, que es el nucleo + miles de programas + montones de documentaciíon. Las más famosas son:
    • Red Hat: Fácil de instalar y excelente para principiantes - http://www.redhat.com
    • SuSE: Una de las mejores, muy completa - http://www.suse.de
    • Debian/GNU: Excelente. - http://www.debian.org
    • Slackware: Para nivel medio-avanzado: http://www.cdrom.com
  • Pueden conseguir las versiones oficiales a 30 - 50 dolares mas o menos o copias -legales gracias a la licencia GNU- entre 2 y 10 dolares. Les recomiendo en Internet:
    • http://www.cheapbytes.com
    • http://www.linuxmall.com
    • Y los sitios oficiales de las distribuciones.
  • También muchas revistas -en su mayoria ibéricas- incluyen distribuciones, dense una vuelta por el kiosko que seguro que algo consiguen.
Bueno, ya con el CD en mano primero sigan estos pasos:
  1. Para instalar Linux y hacer algo bueno, personalmente recomiendo unos 500 MB como mínimo de espacio libre en el disco rígido.
  2. En todas las distribuciones de Linux viene un programa llamado FIPS, que se encarga de rearmar la tabla de particiones de nuestro disco para crear una nueva partición para albelgar a Linux. NOTA: LEER TODA LA DOCUMENTACION DE FIPS CUIDADOSAMENTE - EL AUTOR NO SE HACE CARGO POR LAS PERDIDAS DE INFORMACION QUE EL MAL USO DE FIPS CAUSE EN SU DISCO RIGIDO.
  3. Antes de utilizar FIPS, tenemos que defragmentar el disco, para tener toda la información junta y que nada se borre con FIPS.
  4. Habiendo aprendido FIPS, pasamos a crear 2 particiones: una para albergar a Linux, y una del mismo tamaño que la memoria, que servira de swap (memoria virtual). Por ejemplo, en una maquina con 32 MB de RAM creamos una de 700 MB para Linux y una de 32 MB para la memoria virtual.
  5. Ahora con el CD de la distribución creamos los discos de inicio que contienen el programa de instalación. Esto depende de cada distribución, asi que no me voy a explayar.
  6. Metemos los discos de inicio y rebooteamos el sistema para arrancar el programa de instalación.
  7. Elegimos los paquetes que se adecuan a nuestras necesidades.
  8. Instalamos todo.
  9. A resetear y a disfrutar!
Bueno, con esta ínfima pero concisa introducción a Linux tendrían que poder instalar Linux perfectamente. Ojala que asi lo hagan y esto les sirva. Chau!
